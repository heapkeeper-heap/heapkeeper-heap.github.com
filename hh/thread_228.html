A while back I have assessed the usability of Google App Engine to host our
project once we enter the Hkweb stage [1]. My conclusion then was that the most
painful deficiency of GAE was the inability to handle incoming e-mail. Then, I
proposed we could use an external server to inject incoming mail into GAE via
HTTP requests.

This is exactly what a very kind free service offers [2] -- apparently
without even any advertisements!

If we can wait a bit more, Google itself promises to include this feature in
the next update of GAE [3]. So, it seems GAE might be our target when going on
to the Web stage.

[2] http://www.smtp2web.com/about
[3] http://code.google.com/intl/en-US/appengine/docs/roadmap.html
