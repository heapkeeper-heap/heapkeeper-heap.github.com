I pushed a commit (_auth branch, commit 5806b75, 2010-08-27) where
basic authentication is working, but only with a hardcoded
username/password pair for now. (For the curious, both are 'hk'.)

Later, I'm planning to extend the format of the 'hk.cfg' file to
assign passwords to user nicknames (similarly to how nicknames are
assigned to emails now), and use these to authenticate the users. If
we want, we can even add access control (e.g. setting
none/read-only/read-write access per user per heap).
