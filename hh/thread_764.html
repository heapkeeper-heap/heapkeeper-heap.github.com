We agreed on using the following tags for posts about issues:
- bug: bugs
- feature: new features or improving existing features
- improvement: code improvement that do not provide new features (e.g.
  caching)
- issue: everything else about concrete parts of Hk
- prop: proposals that not about concrete parts of Hk, but about
  coding conventions, procedures, repository usage, workflows, heap
  usage
