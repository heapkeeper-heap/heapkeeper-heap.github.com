I had a look at QUnit, but another tool called JsTestDriver [1] has a
killer feature over it: JsTestDriver can be easily used from the
command line. The code is still executed in a browser, but after you
start the JsTestDriver server and the browser and create a
"JsTestDriver tab" in it, you don't have to care about it any more.
I will integrate JsTestDriver into hk-dev-utils, which will guide you
to do these steps.

I had a look at the other candidate, Rhino [2], but I haven't found a
good tutorial, so I just dropped it and went with JsTestDriver. This
blogger [3] also recommended JsTestDriver instead of other
alternatives.
