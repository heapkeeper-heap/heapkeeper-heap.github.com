Attila Nagy Using the "reviewed" tag ["reviewed" tag, prop, reviewed] <hh/96> <root> (2009-04-17) 
It is already a convention [1], that we tag any posts that require no
further attention from the editors with a tag, and I thought this tag
was [r].

First, Csabi noted that "reviewed" might be a better word that
"revised". I didn't answer this proposal back then, but now I strongly
agree. Revised seems to suggest that it has also been changed, which
is not necessarily true: the post can be marked "complete" by the
editor the same way it entered the heap.

Then, I looked into the matter, and saw that Csabi already tagged some
posts with the "revised" tag.

So, I propose the replacement of both [r] and [revised] tags with
[reviewed].
